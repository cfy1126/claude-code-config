# 学习项目 — 全面架构分析

系统性阅读项目源码，按主次分步骤输出架构说明文档。

## 执行步骤

### 第一步：项目全貌扫描

1. 列出项目所有源文件（排除 node_modules、.git、dist、__pycache__ 等）
2. 读取 `package.json` 或等效的项目配置文件，确定技术栈和项目类型

### 第二步：按主次逐层阅读

严格按以下优先级顺序阅读源码，**不要跳过任何层级**：

**第一层：入口与引导**
- 应用入口文件（main.js / index.js / App.vue / main.ts 等）
- 公共配置（public-path、环境变量、构建配置）

**第二层：核心基础设施**
- 路由配置
- 状态管理（Store / Context）
- 工具函数和桥接层
- API / 请求封装

**第三层：布局与导航**
- 根组件和布局组件
- 页面注册表 / 导航配置

**第四层：业务页面**
- 按功能模块逐个阅读页面组件
- 每个页面的子组件

**第五层：样式与资源**
- 全局样式、主题配置
- 静态资源

### 第三步：生成架构文档

按以下结构输出分析结果，**所有代码引用必须标注文件路径**：

```markdown
# [项目名] 项目架构说明

## 一、项目定位
（技术栈、项目类型、一句话定位）

## 二、核心架构（按主次排列）

### 1. [最核心的机制] — `文件路径`
（解释关键逻辑）

### 2. [次核心的机制] — `文件路径`
（解释关键逻辑）

...

## 三、业务页面

### N. [页面名] — `文件路径`
（功能描述、组件职责表、数据流）

## 四、数据流总结
（ASCII 图示或文字链路）

## 五、文件结构
（树形目录）
```

**编写原则**：
- 每个模块标题标注对应的文件路径
- 使用表格列出函数/组件/配置的职责
- 数据流用简洁的 ASCII 图示
- 不要堆砌代码，用自然语言解释"为什么"和"怎么运转"

### 第四步：保存文档

1. 确认项目 `doc/` 目录存在，不存在则创建
2. 文件名格式：`doc/project-YYYY-MM-DD.md`（取当天日期）
3. 如果当天文件已存在，追加后缀：`doc/project-YYYY-MM-DD-v2.md`
4. 输出文件路径确认
