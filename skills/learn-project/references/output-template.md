# 输出文档模板参考

此文件为 `learn-project` Skill 的输出结构参考，展示最终文档应包含的章节。

## 推荐章节结构

### 必选章节

1. **项目定位** — 技术栈、项目类型、一句话定位
2. **核心架构** — 按重要程度降序排列，每个模块标注文件路径
3. **业务页面** — 每个页面/模块的功能说明、组件职责
4. **数据流总结** — 系统级数据流动路径
5. **文件结构** — 项目目录树

### 按需章节

- **主应用集成机制** — 仅微前端/插件项目需要
- **API 接口对照** — 仅涉及外部接口的项目需要
- **状态管理** — 仅使用全局状态管理的项目需要
- **构建与部署** — 仅构建流程有特殊配置时需要

## 格式规范

- 模块标题格式：`### N. [模块名] — \`文件路径\``
- 函数/组件职责用表格：`| 名称 | 职责 |`
- 数据流用缩进式文本链路，不用复杂图表
- 文件结构用标准树形展示
